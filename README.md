## PT-BR
# PRIINT

**Projeto de Extensão - Universidade Estadual de Ponta Grossa (UEPG)**

## Sobre o Projeto
O PRIINT é uma plataforma educacional desenvolvida sob a arquitetura "Accessibility First". O objetivo do sistema é adaptar dinamicamente a interface gráfica e a metodologia de entrega de conteúdo (chunking, flashcards, textos longos) com base no perfil cognitivo, sensorial e de retenção de cada usuário.

## Tecnologias Utilizadas
* **Front-end:** HTML5, CSS3, TypeScript (Tipagem estática e lógica de interface).
* **Back-end e IA:** Python e TypeScript (Infraestrutura de software e processamento de dados).

## Metodologia de Desenvolvimento
O projeto adota a abordagem **F.G.F.BS**, priorizando a validação visual e estrutural antes da lógica de servidor.

| Fase | Descrição | Status Atual |
| :--- | :--- | :--- |
| **1. Fluxogramas** | Mapeamento lógico de rotas e regras de acessibilidade. | Concluído |
| **2. GitHub Repositories** | Versionamento, documentação e definição de licença. | Concluído |
| **3. Front-End** | Desenvolvimento da UI adaptativa e questionário de onboarding. | Em andamento |
| **4. Back-End & Software** | Algoritmos de processamento de perfil e banco de dados. | Pendente |

## Funcionalidades de Acessibilidade
* **Adaptação Sensorial:** Suporte a TTS (Text-to-Speech) nativo e legendas dinâmicas.
* **Adaptação Cognitiva:** Ajuste de layout para redução de estímulos (TDAH/TEA) e fontes especializadas (Dislexia).
* **Adaptação de Retenção:** Algoritmo de personalização de material de estudo com base no tempo de foco do usuário.

## EN-US
#PRIINT

**Extension Project - State University of Ponta Grossa(UEPG)**

## About the Project
PRIINT is an educational platform developped with the "Acessibility First" architecture. The platform's goal is to adapt dinamically adapta the graphical interface and study methodology(chunking, flashcards and long tets) based on the cognitive, sensorial and retention profile of each user.

## Used technologies
* **Front-end:** HTML5, CSS3 and TypeScript(Estetic typage, interface logic)
* **Back-end and AI:** Python and TypeScript(Software infrastructure and data proccessing).

## Development Methodology
The project uses the **F.G.F.BS** approach, prioritizing visual and structural validation before server logic.

| Phase | Description | Current Status |
| :--- | :--- | :--- |
| **1. Flowcharts** | Logical mapping of routes and accessibility rules. | Completed |
| **2. GitHub Repositories** | Versioning, documentation, and license definition. | Completed |
| **3. Front-End** | Development of adaptive UI and onboarding questionnaire. | In progress |
| **4. Back-End & Software** | Profile processing algorithms and database. | Pending |

## Accessibility Features
* **Sensory Adaptation:** Support for native TTS (Text-to-Speech) and dynamic subtitles.
* **Cognitive Adaptation:** Layout adjustment to reduce stimuli (ADHD/ASD) and specialized fonts (Dyslexia).
* **Retention Adaptation:** Study material personalization algorithm based on the user's focus time.
